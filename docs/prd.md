# 1. 제품 개요

## 1.1 비전

본 플랫폼은 **시스템 아키텍트와 제품 기획자**가 RTL 구현 이전 단계에서
SoC 아키텍처 옵션을 탐색할 수 있도록,
아키텍처 후보를 자동 생성하고 **성능, 전력, 면적(PPA)** 을 추정한다.

플랫폼은 내부 IP 카탈로그 데이터를 활용하여 다음을 수행한다.

- SoC 아키텍처 후보 생성
- PPA 추정
- 파레토 분석 기반 아키텍처 비교
- 고객 제안용 아키텍처 리포트 생성

본 플랫폼은 **초기 아키텍처 탐색 단계**를 대상으로 하며,
설계 공간이 크고 빠른 의사결정이 필요한 상황에 최적화된다.

컴포넌트 모델링

---

Core Die 미모델링
Base Die 모델링
IO Die 모델링

DRAM의 **core die는 블랙박스**로 간주하며,
외부 인터페이스 특성만 모델링한다.

HBM 모델 정의 (v1):

| 항목 | 정의 |
| --- | --- |
| 입력 파라미터 | HBM 스택 수, stack당 pin 수, pin당 데이터율(Gbps), 인터페이스 주파수, 버스 폭, 읽기/쓰기 비율, 목표 대역폭 |
| 가정 | 온도/전압 corner는 PVT 기본 corner 1세트로 고정, DRAM core 동작은 공급사 스펙 기반 black-box 근사 |
| 출력 지표 | 유효 대역폭, 대역폭 활용률, 인터페이스 전력, 메모리 관련 면적 추정 |
| 미포함 항목 | DRAM core die 내부 타이밍/회로 상세, 패키지 SI/PI, 열-전력 연동 해석 |

# 4. 성공 지표

검증 기준선:

- Ground truth는 실리콘 실측 데이터, RTL 시뮬레이션 결과,
  그리고 기준 EDA 툴 출력의 결합으로 정의한다.
- 정확도 지표는 시장 세그먼트 및 공정 노드별 사전 정의된
  golden dataset에서 평가한다.

지표 목표

---

- PPA 순위 정확도 >= 90%
- 전력 추정 오차 +-15%
- 면적 추정 오차 +-10%
- 아키텍처 생성 시간 < 10초 (p95, 최대 200개 후보)

측정 방법 및 합격 기준:

- 평가 데이터셋: 시장 세그먼트별 최소 30개 설계 포인트(전체 최소 90개).
- PPA 순위 정확도: 동일 세그먼트 내 쌍대 비교(pairwise) 일치율.
  `accuracy = (일치한 쌍 수) / (전체 쌍 수)`.
- 전력/면적 오차: MAPE(Mean Absolute Percentage Error) 기준.
  `MAPE = mean(abs(pred - gt) / gt) * 100`.
- 합격 기준: 각 세그먼트(MEMORY/MOBILE/PHYSICAL AI)별로 기준을 개별 충족해야 하며,
  전체 평균 충족만으로는 통과로 간주하지 않는다.
- 통계 신뢰도: 95% bootstrap CI의 상한(오차 지표) 또는 하한(정확도 지표)도
  동일 기준을 만족해야 한다.

# 11. 데이터 관리

데이터베이스:

PostgreSQL

주요 테이블:

- ip_block
- ip_version
- ip_variant
- ppa_model
- rule_set
- candidate
- result_snapshot

재현성 및 추적성 요구사항:

- `result_snapshot`에는 아래 메타데이터를 필수 저장한다.
- 모델 버전(`ppa_model_version`), 룰셋 버전(`rule_set_version`), IP 카탈로그 버전
- 입력 요구사항 정규화 JSON 및 입력 해시(`input_hash`)
- 실행 환경 정보(앱 버전, solver 버전, 실행 시각 UTC, 실행자 역할)
- 동일 `input_hash + model/rule_set version` 조합에서 결과 재현 가능해야 한다.

# 12. 보안

민감한 내부 데이터:

- IP 파라미터
- PPA 모델

접근 정책 (RBAC):

- 시스템 아키텍트: 내부 파라미터 및 모델 상세 전체 접근
- 제품 기획자: 제안/리포트 수준 접근만 허용, 기밀 모델 파라미터 마스킹
- IP 오너: 본인 소유 IP 항목 및 모델 아티팩트 편집 접근

비아키텍트 사용자에게 내보내는 리포트는
기밀 파라미터를 반드시 숨겨야 한다.

보안 구현 요구사항 (v1 필수):

- 인증: 사내 SSO(OIDC 또는 SAML) 연동 필수, 로컬 계정 로그인 비허용.
- 인가: API 계층 RBAC 강제, DB 직접 접근은 서비스 계정만 허용.
- 암호화: 전송 구간 TLS 1.2+, 저장 구간 AES-256(디스크/백업) 적용.
- 감사 로그: 로그인/권한 변경/모델 조회/모델 수정/리포트 내보내기 이벤트를
  불변 로그로 1년 보관.
- 마스킹 정책: 제품 기획자 리포트에는 절대값 파라미터 대신 범주/등급만 노출.

권한 체크 지점:

- UI 메뉴 노출 제어
- API endpoint 권한 체크
- 리포트 생성 시 필드 단위 마스킹

# 13. 비기능 요구사항 (NFR)

- 성능: p95 응답시간 10초 이하(200 후보), p99 20초 이하.
- 처리량: 동시 사용자 20명 기준, 탐색 요청 처리 실패율 1% 이하.
- 가용성: 월간 99.5% 이상(계획 점검 제외).
- 복구: 장애 시 RPO 24시간, RTO 4시간.
- 신뢰성: solver 실패 시 최대 2회 자동 재시도 후 실패 사유 코드 반환.
- 관측성: latency/error/queue length 메트릭과 감사 이벤트 대시보드 제공.
