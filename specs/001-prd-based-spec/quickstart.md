# Quickstart: PRD 기반 SoC 아키텍처 탐색 및 PPA 추정

## 목적
이 문서는 본 기능의 핵심 시나리오(P1/P2/P3)와 헌법 게이트를 빠르게 검증하기 위한 실행 순서를 제공한다.

## 사전 조건
- 활성 브랜치: `001-prd-based-spec`
- 스펙/플랜/연구/데이터모델/계약 문서 최신화
- 테스트 데이터셋(세그먼트별 최소 30개 설계 포인트)

## 1) 품질 게이트 검증
1. 정적 분석/포맷/린트 실행
2. PR 리뷰 체크리스트에 요구사항 추적 링크 첨부
3. 변경 동작과 관련 문서 업데이트 여부 확인

## 2) 테스트 게이트 검증
1. 단위 테스트: 정규화, 후보 필터링, 재시도 로직, 충돌 감지
2. 통합 테스트: 요구사항 입력 -> 후보 생성 -> 비교 -> 리포트
3. 계약 테스트: `contracts/exploration-api.yaml` 스키마 적합성 검증
4. 회귀 테스트: 동일 입력+동일 버전 100회 반복 실행, 결과 불일치율 0% 확인

## 3) UX 일관성 검증
1. 모든 주요 화면에서 로딩/빈 결과/오류/성공 상태 확인
2. 용어 일관성 검증(후보, 파레토, 예산 초과)
3. 기본 결과 노출이 상위 5개로 고정되는지 확인

## 4) 성능/신뢰성 검증
1. 최대 200개 후보 조건에서 p95 <= 10초, p99 <= 20초 검증
2. 동시 사용자 20명 부하에서 실패율 <= 1% 검증
3. 실패 시 자동 재시도 3회 동작 및 최종 실패 코드 노출 검증

## 5) 보안/권한 및 충돌 처리 검증
1. 역할별 접근 제어 동작 확인
2. 리포트에 절대값 수치 포함 여부 확인(마스킹 없음)
3. 동일 IP 항목 동시 수정 시 409 충돌 및 저장 차단 확인

## 6) 완료 기준
- 스펙의 SC-001 ~ SC-009가 모두 통과
- 헌법 게이트(Code Quality, Testing, UX, Performance, Evidence) 모두 통과
- 릴리스 증거 패키지(테스트 리포트, 성능 리포트, 계약 검증 결과) 보관
